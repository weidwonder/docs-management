# 文档整合工作流

## 触发条件

当发现以下情况时，应进行文档整合：
1. 存在多个位置的重复文档
2. 文档散落在非标准位置
3. 完成重大功能开发后

## 整合步骤

```
Step 1: 扫描文档
- 列出所有 .md 文件
- 标记位置、行数、主题

Step 2: 分析重复
- 对比内容相似的文档
- 确定保留哪个版本

Step 3: 制定计划
- 哪些文档移动到哪里
- 哪些文档需要合并
- 哪些文档需要删除

Step 4: 执行整合
- 创建新目录（如需要）
- 移动/重命名文档
- 删除重复/过时文档

Step 5: 更新导航
- 更新 CLAUDE.md 文档导航
- 检查所有内部链接
```

## 整合决策矩阵

| 情况 | 决策 |
|-----|------|
| 框架级文档 vs 模块级文档 | 分开存放：`docs/` vs `docs/{module}/` |
| 内容重复的文档 | 合并到更完整的版本，删除另一个 |
| 过程性文档（计划、调试记录） | 完成后删除或归档到 `docs/plans/` |
| 子目录中的通用文档 | 移动到 `docs/` 或 `docs/guides/` |

---

## 检查清单

### 创建新文档前

- [ ] 确认没有现有文档覆盖该主题
- [ ] 确定正确的存放目录
- [ ] 使用标准命名规范

### 完成文档后

- [ ] 在 CLAUDE.md 中添加导航链接
- [ ] 检查内部链接是否正确
- [ ] 确认行数在建议范围内

### 定期维护

- [ ] 检查文档与代码是否同步
- [ ] 清理过时的计划文档
- [ ] 合并重复内容
